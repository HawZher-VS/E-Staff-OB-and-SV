Guidelines · 16

Do's & Don'ts

Core rules for VX v2. Follow these to keep the UI consistent across all modules.

check_circleDo
  • Place KPI card icon tile in the top-right corner, never left of the value.
  • Use ALL CAPS 11px for data table column headers with 0.08em letter-spacing.
  • Use underline-style tabs (2px brand border-bottom) inside data panels.
  • Use dot + coloured text for status — no pill background in tables.
  • Use breadcrumb in the page header — parent grey, current page bold.
  • Use module-specific icon tile colours consistently across sidebar, charts, and tables.
  • Use #5865F2 (brand) as the single primary accent — chart bars, active nav, focus rings, avatars.
  • Use #2DCE89 for positive monetary values and approved states.
cancelDon't
  • Don't use filled pill tabs inside table panels — underline only.
  • Don't use page-background colour (#f0f2f5) inside a card surface.
  • Don't mix button sizes in the same action row — pick one size.
  • Don't use ad-hoc hex values outside the defined token palette.
  • Don't display ALL CAPS names — always apply Title Case at render time.
  • Don't use a full-screen modal for a simple lookup — use a combobox dropdown.
  • Don't render status without both a dot and coloured text — colour alone is insufficient for accessibility.
  • Don't use pagination when search is available.
